# Plan: Silence Translated Markdown Warning

Here is why that warning occurs and how we can silence it forever:

---

## 🔍 The Root Cause:
When a user's browser requests `en-US` locale, Chainlit tries to find a translation file named `chainlit_en-US.md` in the application directory. When it can't find one, it logs the warning:
`Translated markdown file for en-US not found. Defaulting to chainlit.md.`

---

## 🛠️ The Permanent Fix:
We can create a symbolic link `app/chainlit_en-US.md` pointing directly to `app/chainlit.md`:
```bash
ln -sf chainlit.md app/chainlit_en-US.md
```
This is a zero-byte solution that git natively tracks. Chainlit will find the requested translation file, load it instantly, and **silence the warning forever!**
